## Project summary
Static website hosting photos/descriptions of estate sale & auction items (antiques, household, farm, construction/tools, machinery) to support advertising on Facebook Marketplace, Nextdoor, and OfferUp ahead of a Kansas auction house sale. No login, no backend, no stored visitor data.

## Decisions locked in
- **Data schema:** `items.json` — fields: `id`, `lot`, `title`, `category`, `status`, `shortDescription`, `description`, `notes`, `dimensions`, `condition`, `price`, `photos[]`, `tags[]`. Now has 18 sample items across all 6 categories (expanded from the original 5 using terms pulled from a real Kansas-auction word cloud).
- **Categories:** Antiques, Household, Crafts, Farm, Construction & Tools, Machinery. (`Crafts` added per stakeholder feedback; no items tagged with it yet, so it currently shows the built-in empty-category state on `items.html` — expected, not a bug.)
- **No sign-in / no backend / no stored personal data** — confirmed hard requirement. Site is explicitly personal/local, not a business — footers deliberately carry no `©` (removed), and the "no accounts, no tracking" note now lives only on `contact.html` instead of repeating on every page.
- **Stack:** Static HTML/CSS/JS, no server, **styled with a full Tailwind CSS build** (Node + `tailwindcss` CLI). Build step is source-only — the shipped site is still 100% static files; Cloudflare Pages/Netlify run `npm run build` automatically on push, so end users and even the site owner (once deployed) never touch the build step directly.
- **Site structure (built):** `index.html` (hero + featured carousel + category tiles), `items.html` (search + category filter + detail modal), `contact.html` (mailto/phone + FAQ accordion — no form). Shared `js/app.js` and Tailwind-compiled `css/style.css`. Location copy currently reads "Fredonia, Kansas area."
- **Repository:** git-initialized, pushed to **both** `github.com/clanielh/Family-Farm-Selling-Site` and `gitlab.com/clanielh/family-farm-selling-site` (histories merged — GitLab's own `.gitlab-ci.yml` SAST/Secret Detection config is preserved on both remotes now). Push to both when committing going forward unless told otherwise.
- **Component library (built, checklist.design-informed):** carousel (manual + autoplay, dots, pause, swipe, reduced-motion aware), sticky nav with scroll shadow + mobile menu, footer with utility links, item cards, live search bar, tooltips (glossary terms like "starting bid"), FAQ accordion, skeleton-card loading states, accessible modal/lightbox.
- **Item card visual style — decided.** Ran the owner's own design brief through Figma Make (a code-generation design tool) to get fresh concepts beyond the original 4 (Ledger Row/Catalog Card/Tag Card/Spec Sheet). Figma returned 3 new "physical print metaphor" concepts (Perforated Ticket, Giant Inventory Tag, Framed Placard); owner picked **Giant Inventory Tag** — vertical shipping-tag layout, hole-punch circle, photo inset mid-card, price/status/action in a footer band. Implemented in `.item-card` (`src/input.css`) + `buildCard()` (`js/app.js`) on `items.html`, replacing the old simple placeholder card. Sharp corners (no `rounded-*`) kept deliberately, as part of this card's tag/ticket identity — a departure from the softer `rounded-md` used elsewhere on the site, and worth knowing about if it ever looks like an inconsistency rather than a choice. `items-grid`'s minmax bumped 260px→300px to fit it; this card is taller/lower-density than the old one (expected tradeoff of the "massive tap target" brief, not a bug). See `memory.md` for the specific adaptations made porting the Figma React/Tailwind-v4 code into this vanilla/Tailwind-v3 site (price-string parsing, dropped sepia/grayscale photo filter, dropped on-photo lot/status badges, "Bid Now"→"View details" copy fix).
  - **Follow-up fix (post-launch feedback):** the hole-punch circle was clipping into the category text (card's top padding wasn't quite enough clearance below it — bumped `pt-10`→`pt-14`). Also rebalanced the header hierarchy per owner feedback: category+lot were originally the dominant element (huge 32px lot number) with the title secondary — now category+lot share one small line (`Category · LOT ID`, ~11px) and the **title** is the largest text on the card (bumped to `text-2xl`/bold), since the item name should be the focal point, not the lot number.
- **Light/dark mode (built):** icon+text toggle in the nav (`.theme-toggle`, next to the mobile hamburger inside a new `.nav-actions` wrapper), defaults to light for every first-time visitor regardless of device OS setting, persists the choice in `localStorage['theme']`. Architecture: Tailwind `darkMode:'class'` + CSS variables (`:root`/`.dark` in `src/input.css`) so existing `@apply bg-paper text-ink` usage flips automatically. Two color-token tiers: **reactive** (`paper`, `paper-deep`, `ink`, `ink-soft`, `surface`, `barn`, `brass`, `sage`, `hairline` — page chrome) vs. **fixed** (`cream`, `midnight` — badges/tooltip/scrim/footer band, which stay a dark-chip-on-photo look in both themes on purpose) vs. **literal hex** (status tag colors — semantic, don't change with theme at all). See `memory.md` (2026-07-18 entry) for the full reasoning if this needs revisiting.
- **Mobile-responsive pass (built):** touch targets on `.modal-close`/`.modal-photo-nav`/`.carousel-pause`/`.carousel-arrow` bumped to 44px (was 30–36px); modal goes full-bleed (no rounded corners/side padding) under 480px; carousel photo height and hero/section padding trimmed under 480px. Most of the layout (item/category/footer grids, nav, contact page) was already responsive via CSS Grid `auto-fit` and existing breakpoints, so this was a gap-filling pass, not a rebuild.
- **Hosting recommendation:** **Cloudflare Pages**, connected to a GitHub repo — free, unlimited bandwidth, strongest DDoS/edge protection of the free options, automatic HTTPS, and pairs cleanly with the Tailwind build step (Cloudflare runs `npm run build` on every push). **Netlify** is the fallback if drag-and-drop-without-git is preferred over connecting GitHub. A GitHub (or GitLab) repo is required to get Cloudflare's automatic build-on-push — direct upload without git would mean running `npm run build` locally and re-uploading manually every time.
  - **Confirmed Cloudflare Pages dashboard settings:** Framework preset: `None` · Build command: `npm run build` · Build output directory: `/` (site files live at repo root, not a separate `dist/` folder) · No environment variables needed (no secrets, fully static).
  - **Node version gotcha (handled):** Cloudflare defaults to an old pinned Node version unless overridden. Fixed by adding `.node-version` (set to `22`) to the repo root.
  - `node_modules`, `.git`, `.DS_Store` are auto-excluded from Cloudflare's deploy regardless of `.gitignore` — no extra cleanup needed before pushing.
  - Final pick and actual deploy: *not yet done* — next concrete step.

## Open decisions
- **Modal ("view details") redesign — pending stakeholder feedback.** Two more Figma Make batches were reviewed (6 more concepts: Inspector's Tag, Archival Folder, Bill of Sale/Ledger Entry, Public Notice, Ruled Index Card, Quadrant Lot Box — none of the 9 total concepts across all 3 rounds were actually designed as a modal, since Figma doesn't know a separate modal exists; the plan is to repurpose one of the rejected card concepts for it instead). Claude's ranked recommendation, for when this comes back: **Quadrant Lot Box** (4-zone grid built around "impossible to miss any data," room to add dimensions/condition/notes) or **Inspector's Tag** (clean image-one-side/facts-the-other split, photo gets full-height room) as the two strongest fits — both have proper room for a real photo area (needed for the existing multi-photo slideshow's prev/next arrows), unlike Bill of Sale/Ruled Index Card which treat the photo as a small rotated "attachment." Owner chose to hold for peer/stakeholder feedback across all 6 rather than commit now, same process as the grid card decision.
